# Argus Recovery Requirement Matrix

This matrix is the recovery gate for the server-mode implementation. It is sourced from the actual Argus docs copied into `/Users/mike/.openclaw/workspace/` and from no host-topology documents.

| Source doc | Requirement | Implementation evidence |
| --- | --- | --- |
| `notes/argus-server-requirements.md` section 1 | Argus publishes well-formed, source-grounded, embedded news reports into Subspace; subscribers perform interpretation and filtering. | Package payloads keep source/provenance fields and canonical `supplied_embeddings`; docs describe downstream receivers as separate from Argus publishing. |
| `notes/argus-server-requirements.md` section 3.1 | Argus publishes to the approved Subspace target using its configured publisher identity after live publishing approval. | `PublishConfig` uses `subspace_endpoint`, `subspace_websocket_path`, `subspace_agent_id`, and env-only `ARGUS_SUBSPACE_SESSION_TOKEN`; active mode remains gated by `publish.state` and `live_approval`. |

| `notes/argus-server-requirements.md` section 8.1 | Argus must not require receiver-side components, local subscribers, or desktop runtime to publish. | The runtime publish path no longer calls receiver-side socket/API helpers; legacy publish topology fields are rejected by config loading. |
